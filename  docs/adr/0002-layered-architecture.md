# 2. Шарова архітектура

**Дата:** 2025-10-XX  
**Статус:** Прийнято

## Рішення
Ми використовуємо суворий поділ на шари:

1. **API Layer (`api/`)**:
   - Приймає HTTP запити.
   - Валідує вхідні DTO.
   - Викликає Service.
   - Повертає JSON відповідь.
   - *Заборонено:* Звертатися напряму в БД або виконувати бізнес-логіку.

2. **Service Layer (`service/`)**:
   - Містить сценарії використання (Use Cases).
   - Керує транзакціями.
   - Викликає Domain.
   - Працює з БД (або викликає репозиторії).
   - *Заборонено:* Знати про HTTP (request/response об'єкти).

3. **Domain Layer (`domain/`)**:
   - Чисті сутності та бізнес-правила.
   - Не залежить ні від чого (ні від БД, ні від Flask).

## Напрямок залежностей
API -> Service -> Domain
