# 1. Вибір архітектурного стилю

**Дата:** 2025-10-XX  
**Статус:** Прийнято

## Контекст
Проєкт "Music Platform" знаходиться на стадії MVP. Розробка ведеться невеликою командою. Необхідно забезпечити швидку розробку, але зберегти можливість масштабування.

## Рішення
Обрано стиль **Модульний Моноліт**.
Весь код знаходиться в одному репозиторії та деплоїться як один процес, але логічно розділений на модулі (Contexts).

## Наслідки
* **Позитивні:** Простий деплой, легкий рефакторинг, відсутність мережевих затримок між модулями.
* **Негативні:** При значному зростанні кодової бази час збірки може збільшитись.

## Умови перегляду
Перехід на мікросервіси розглянемо, коли:
1. Над різними модулями працюватимуть різні команди ( > 2 команд).
2. Навантаження на модуль Catalog перевищить можливості масштабування моноліту.
